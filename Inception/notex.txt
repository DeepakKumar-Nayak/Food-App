What is CDN
------------
it stand for Content Delivery Network. 
CDN ka full form hota hai Content Delivery Network. Yeh duniya bhar mein 
phaile hue servers ka ek group (network) hota hai, jo internet par
websites aur data ko aap tak bahot tezi se pahunchane
ka kaam karta hai.

Ek simple example se samjhte hain:
----------------------------------
Maan lijiye aapko Amazon se ek item kharidna hai. Agar Amazon ka sirf 
ek hi warehouse America mein ho, toh India tak delivery aane mein
hafton lag jayenge. Is problem ko solve karne ke liye Amazon ne India 
ke har bade shahar mein apne local warehouses bana rakhe hain. Jab aap order 
karte hain, toh item aapke sabse kareeb wale warehouse se aata hai, jisse delivery fast 
ho jati hai! CDN bilkul yahi kaam websites ke liye karta hai.


Bina CDN ke kya hota hai?
-------------------------
Agar aap India mein baith kar ek aisi website open kar rahe hain jiska main server 
(jise Origin Server kehte hain) US mein hai, toh website ke har photo, video, aur 
page ko US se India travel karke aana padega. Is physical distance ki wajah se 
website load hone mein time lagta hai, jise technical bhasha mein latency kehte hain.


CDN Kaam Kaise Karta Hai?
CDN is loading time ko lagbhag khatam kar deta hai. Yeh 3 steps mein kaam karta hai:

1. Edge Servers (Local Warehouses): CDN providers (jaise Cloudflare, AWS CloudFront, 
ya Akamai) poori duniya ke alag-alag shahron mein apne servers laga ke rakhte hain. 
Inhe Edge Servers ya Point of Presence (PoP) kehte hain.

2. Caching (Copy Save Karna): Jab koi user pehli baar website open karta hai, 
toh CDN us website ka "static data" (jaise images, videos, fonts, aur design f
iles) main server se laakar apne Edge Servers par copy karke save kar leta hai. 
Is process ko caching kehte hain.

3.Fast Delivery: Jab aap ya koi aur aapke aas-paas se us website ko dobara open karta hai, 
toh request US wale main server tak jaati hi nahi hai. Aapke sabse kareeb wale server 
(maan lijiye Mumbai ya Delhi) se hi data turant aapke phone par aa jata hai.

Reason Cross-Origin you will see in React Cdn Links:
----------------------------------------------------
React ke CDN links (jo aksar react, react-dom ke liye use hote hain) mein 
crossorigin ka hona aur uske peeche ka main reason "Error Reporting" hai.

Jab aap React ko bina build tools (jaise Vite ya Webpack) ke seedha CDN se 
load karte hain, toh aap crossorigin attribute ka istemal isliye karte 
hain taaki Error Handling sahi se kaam kar sake.

Jab aap crossorigin attribute ka use karte hain, toh aap browser ko batate 
hain ki "main is cross-origin script ko fetch karne ke liye CORS ka istemal 
kar raha hoon."

Bina crossorigin ke: Agar React ki kisi file mein koi runtime error aata hai, 
toh browser security reasons (CORS policy) ki wajah se aapke console mein 
error ki detail nahi dikhata. Woh sirf "Script Error" likha hua dikhayega, 
jisse aapko yeh nahi pata chalega ki error kis line ya kis file mein hai.

crossorigin ke saath: Browser ko permission mil jaati hai ki woh error ki 
poori information (file ka naam, line number, column number) aapke browser 
console mein dikha sake. Development ke dauran yeh bahut zaroori hai.

root.render() behind the scenes karta kya hai .??
-------------------------------------------------
Haan, bilkul sahi samjhe aap! root.render(heading) ka kaam hi yahi hai ki 
woh aapke React Element (jo ki ek JavaScript object hota hai) ko asli 
HTML mein convert karke browser ke DOM (Document Object Model) mein 
inject (render) kare.

root.render() kya karta hai?
root.render() woh bridge hai jo React ki Virtual DOM se Real DOM mein data transfer karta hai.

Iske steps yeh hote hain:
--------------------------

Virtual DOM comparison: React check karta hai ki kya pahle se wahan koi 
content hai ya nahi.

Conversion: React us JavaScript object (heading) ko browser ke 
samajhne layak HTML element (<h1>Hello World</h1>) mein convert karta hai.

DOM Injection: Yeh us element ko aapke HTML ke us specific div mein daal 
deta hai jise aapne root banaya tha (jaise <div id="root"></div>).


Kya hota agar aap root.render() nahi karte?
Agar aap sirf const heading = <h1>Hello World</h1>; likhenge, 
toh woh sirf ek JavaScript variable mein ek object bankar reh jayega. 
Browser ko pata hi nahi chalega ki use screen par kya aur kahan dikhana hai.
