With .parcel-cache: Parcel cache mein check karta hai aur dekhta

Cache Busting and Content Hashing:
-----------------------------------
Bhai, Cache Busting aur Content Hashing ka logic samajhna bahut zaroori hai, kyunki ye industry ka standard tareeka hai users ko hamesha "Fresh Code" dikhane ka.Chaliye isse ek real-life problem se samajhte hain:

1. Problem: Browser ki purani aadat (Caching)
Jab koi user aapki website pehli baar visit karta hai, toh browser fast load karne ke liye index.js aur style.css ko apne paas save (cache) kar leta hai.

Ab maan lo aapne code mein ek naya feature dala aur wapas deploy kiya. Lekin user ka browser sochega, "Mere paas toh pehle se hi index.js rakhi hai, main server se nayi file kyun mangun?" Isse user ko purani website hi dikhti rehti hai.

2. Solution: Content Hashing
Yahan Parcel aur Vite jaise bundlers jaadu dikhate hain. Wo aapki 
file ka naam badal kar usmein ek Unique Hash jod dete hain, jaise:

Purana: index.js
Naya: index.8472fb.js

Ye Hash kaise banta hai? Bundler aapke code ke content ko read karta hai.
Agar content same hai -> Hash same rahega.
Agar aapne ek semicolon (;) bhi badla -> Hash badal jayega (e.g., index.9912ac.js).

3. Cache Busting (The Result)
Jab aap naya build bhejte ho, toh HTML mein script ka naam badal chuka hota hai (index.9912ac.js). Browser jab ye naya naam dekhta hai, toh wo confuse ho jata hai: "Ye toh nayi file hai, ye meri memory mein nahi hai!" Wo majbooran server se nayi file download karta hai. Isi process ko hum Cache Busting kehte hain (yani purane cache ko tod dena).

4. Interviewer ko kya bolna hai?
"Content Hashing is a technique where bundlers append a unique string (hash) 
to the filename based on the file's content. This enables Cache Busting.

Since browsers cache static assets to improve performance, they might not fetch 
updated files if the filename remains the same. By changing the filename whenever 
the content changes, we force the browser to download the latest version of the 
application, ensuring the user never sees an outdated version."

Summary in a Nutshell:
Content Hashing: File ke content ke hisaab se naam dena.

Cache Busting: Naye naam ki wajah se browser ko naya code download karne par majboor karna.

Bhai, ek mazaak wali baat: Agar aapne dist folder mein manually filename badal 
diya bina HTML update kiye, toh aapki site load hi nahi hogi! Bundler ye dono 
kaam (hash badalna aur HTML mein link update karna) ek saath karta hai.
