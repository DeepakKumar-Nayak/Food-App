Ek Sample package.json ka Structure

{
  "name": "my-awesome-project",
  "version": "1.0.0",
  "description": "Mera pehla project",
  "scripts": {
    "start": "parcel index.html",
    "build": "parcel build index.html"
  },
  "dependencies": {
    "react": "^18.2.0",
    "react-dom": "^18.2.0"
  },
  "devDependencies": {
    "parcel": "^2.9.0"
  }
}


. scripts: Yahan aap shortcut commands define karte hain (jaise npm start likhte hi Parcel shuru ho jaye).

. dependencies: Wo library jo app ko chalne ke liye hamesha chahiye.

. devDependencies: Wo tools jo sirf coding ke waqt chahiye (jaise Parcel), lekin app deploy hone ke baad unki zaroorat nahi hoti.

Type of Dependency:
-------------------
Normal Dependency: Ye wo packages hain jo aapki website ko Browser mein chalne ke liye chahiye hi chahiye. Inke bina aapki app crash ho jayegi.
    1. Examples: React, Redux, Axios, Framer Motion.
    2. Logic: Jab aap final build banate ho, toh bundler inka code uthakar dist folder mein daal deta hai.

Dev Dependencies: Ye wo tools hain jo sirf Aapko (Developer ko) chahiye code likhte waqt, testing karte waqt,  ya build banate waqt. Browser ko inke baare mein kuch nahi pata.
    . Examples: Parcel, Vite, Jest (testing), ESLint (code formatting), Babel.
    . Logic: Inka kaam sirf dist folder banana hai. Ek baar dist ban gaya, toh inka kaam khatam. Ye final bundle ka hissa nahi bante.

Transitive Depedency:
--------------------
aggar hamne ek Dev Dependency install kiya for exmaple parcel 
parcel ke paas bhi apna khud ka depency hoga woh bhi kisi ke uper depend hoga esko ham bolte hai Transitive dependency

Aapne Parcel install kiya (ye aapki Direct Dependency hai). Lekin Parcel ko chalne ke liye 
Babel chahiye, aur Babel ko chalne ke liye Access-code chahiye.
kyun ki parcel apne aap khud se sab kcuh nahi kar sakhta usko bhi help chahiye

Aapne toh sirf Parcel manga tha, lekin Parcel apne saath poori "baraat" 
(uski apni dependencies) lekar aaya. Inhi piche se aane wali libraries ko hum Transitive Dependencies kehte hain.

2. English Mein (Technical Interview Version)
"A Transitive Dependency occurs when your project depends on Library A, 
and Library A itself depends on Library B. Even though you didn't explicitly install Library B, 
it becomes a part of your project's node_modules because Library A cannot function without it."

"NPM manages these transitive dependencies by building a Dependency Tree. It ensures 
that all the required sub-packages are fetched and stored so that the primary package (like Parcel) works as expected.

How do you check the Transitive Dependency:
--------------------------------------------
Aap 3 tareekon se check kar sakte ho:

node_modules folder: Aap dekhoge ki aapne sirf parcel dala tha, 
lekin folder mein 100 aur folders aa gaye.

package-lock.json: Is file ko khologe toh wahan har package ke niche ek 
requires ya dependencies ka section hoga. Wohi transitive dependencies hain.

Command: Terminal mein npm list chalao. Ye aapko ek poora tree dikhayega ki kaun kis par depend hai.
